p>Results are split over multiple csv files with each file name matching the pattern "[operation_id]_[count].csv", for example 3094877188788974909_1.csv. The operation_id matches the identifier for the Operation, and the count is a counter used for tracking the number of files written.

p>The CSV file(s) contain the following columns regardless of storage type scanned:
